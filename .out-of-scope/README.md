# Out of scope

One markdown file per deliberately-rejected feature request, named after the rejected idea (e.g. `salesforce-as-first-class-crm.md`).

Each entry should cover:

1. **What's out of scope** — one sentence stating the rejection.
2. **Why** — the trade-off that made us say no (maintenance surface, scope creep, conflicts with another decision, etc.).
3. **Escape hatches** — what users with this need can do instead.
4. **Prior requests** — issue numbers or links that prompted the rejection, so future maintainers can find the precedent.

ADRs in `docs/adr/` document what we _are_ doing; entries here document what we're not — and why we keep saying no.
